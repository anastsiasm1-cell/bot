## Шаг 1: Аренда сервера

1. Зарегистрируйтесь на выбранном хостинге
2. Создайте VPS с Ubuntu 22.04 LTS
3. Выберите минимальный тариф (1 ГБ RAM достаточно)
4. Запишите данные для подключения:
   - **IP-адрес сервера** (например, `123.45.67.89`)
   - **root-пароль** (придёт на email или будет показан при создании)

---

## Шаг 2: Подключение к серверу

### На Windows

1. Скачайте и установите [PuTTY](https://www.putty.org/)
2. Откройте PuTTY
3. В поле "Host Name" введите IP-адрес сервера
4. Нажмите "Open"
5. Введите логин: `root`
6. Введите пароль (символы не отображаются — это нормально)

### На macOS / Linux

Откройте терминал и выполните:

```bash
ssh root@ВАШ_IP_АДРЕС
```

Например:
```bash
ssh root@123.45.67.89
```

Введите пароль при запросе.

## Шаг 3: Установка Docker

Выполните эти команды на сервере (копируйте и вставляйте по одной):

```bash
# Обновление системы
apt update && apt upgrade -y

# Установка необходимых пакетов
apt install -y ca-certificates curl gnupg lsb-release

# Добавление репозитория Docker
mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установка Docker
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Проверка установки
docker --version
docker compose version
```

Если обе команды показали версии — Docker установлен успешно.

## Шаг 4: Создание пользователя deploy

Создаём отдельного пользователя для безопасности:

```bash
# Создание пользователя
adduser deploy

# Следуйте инструкциям:
# - Придумайте пароль
# - Остальные поля можно оставить пустыми (нажимайте Enter)

# Добавление в группу docker
usermod -aG docker deploy

# Добавление в группу sudo (для административных команд)
usermod -aG sudo deploy
```

## Шаг 5: Настройка SSH-ключей

SSH-ключи нужны для автоматического подключения GitHub Actions к серверу.

### Генерация ключей на сервере

```bash
# Переключаемся на пользователя deploy
su - deploy

# Генерируем SSH-ключ
ssh-keygen -t ed25519 -C "github-actions-deploy"

# Нажмите Enter на все вопросы (пароль оставьте пустым)
```

### Настройка авторизации по ключу

```bash
# Создаём папку для ключей (если её нет)
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# Добавляем публичный ключ в authorized_keys
cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# Выводим приватный ключ (он понадобится для GitHub)
cat ~/.ssh/id_ed25519
```

**ВАЖНО:** Скопируйте содержимое приватного ключа (от `-----BEGIN` до `-----END` включительно). Он понадобится на шаге 9.

## Шаг 6: Клонирование проекта

Оставаясь под пользователем `deploy`:

```bash
# Установка git (если не установлен)
sudo apt install -y git

# Переход в домашнюю директорию
cd ~

# Клонирование репозитория (замените URL на ваш)
git clone https://github.com/ВАШ_USERNAME/ВАШ_РЕПОЗИТОРИЙ.git

# Переход в папку проекта
cd ВАШ_РЕПОЗИТОРИЙ

# Установка make (если не установлен)
sudo apt install -y make
```

## Шаг 7: Настройка переменных окружения

```bash
# Создание файла настроек из шаблона
cp .env.prod.example .env.prod

# Редактирование настроек
nano .env.prod
```

### Что нужно изменить в файле:

| Параметр | Что вписать |
|----------|-------------|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `BOT_USERNAME` | Username бота (без @) |
| `ADMIN_USER_IDS` | Ваш Telegram ID (узнайте у @userinfobot) |
| `POSTGRES_PASSWORD` | Придумайте сложный пароль |
| `REDIS_PASSWORD` | Придумайте сложный пароль |

### Как сохранить в nano:
1. Внесите изменения
2. Нажмите `Ctrl + X`
3. Нажмите `Y` (да, сохранить)
4. Нажмите `Enter`

## Шаг 8: Первый запуск

```bash
# Запуск бота
make prod

# Проверка статуса
make status

# Просмотр логов (Ctrl+C чтобы выйти)
make logs-bot
```

Если бот запустился — отправьте ему команду `/start` в Telegram.

## Шаг 9: Настройка GitHub Actions

Теперь настроим автоматический деплой при каждом пуше в main.

### Добавление секретов в GitHub

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** → **Secrets and variables** → **Actions**
3. Нажмите **New repository secret**
4. Добавьте следующие секреты:

| Имя секрета | Значение |
|-------------|----------|
| `SERVER_HOST` | IP-адрес вашего сервера (например, `123.45.67.89`) |
| `SERVER_USER` | `deploy` |
| `SSH_PRIVATE_KEY` | Приватный ключ из шага 5 (полностью, включая BEGIN и END) |
| `PROJECT_PATH` | Путь к проекту (например, `/home/deploy/aiogram_starter_kit`) |

### Проверка пути к проекту

На сервере выполните:
```bash
pwd
```
Скопируйте результат — это и есть `PROJECT_PATH`.

## Шаг 10: Проверка автодеплоя

### Тестовый деплой

1. Сделайте любое небольшое изменение в коде (например, в README)
2. Закоммитьте и запушьте в main:
   ```bash
   git add .
   git commit -m "Test deploy"
   git push origin main
   ```
3. Откройте вкладку **Actions** в вашем репозитории на GitHub
4. Наблюдайте за выполнением workflow

### Признаки успешного деплоя

- Зелёная галочка на вкладке Actions
- Бот отвечает на команды
- Логи показывают перезапуск: `make ci-logs` (на сервере)

## Частые проблемы

### "Permission denied" при подключении

**Причина:** SSH-ключ не настроен правильно.

**Решение:**
```bash
# На сервере под root
cat /home/deploy/.ssh/id_ed25519.pub >> /home/deploy/.ssh/authorized_keys
chmod 600 /home/deploy/.ssh/authorized_keys
chown deploy:deploy /home/deploy/.ssh/authorized_keys
```

### "Connection timed out"

**Причина:** Firewall блокирует подключения.

**Решение:**
```bash
# На сервере под root
ufw allow 22/tcp
ufw allow OpenSSH
```

### Бот не отвечает после деплоя

**Решение:**
```bash
# На сервере
cd ~/ВАШ_РЕПОЗИТОРИЙ
make ci-logs
```
Проверьте логи на наличие ошибок.

### "CHANGE_ME" error в CI

**Причина:** Не изменены пароли по умолчанию в `.env.prod`.

**Решение:** Отредактируйте `.env.prod` и замените все пароли.

### Docker команды требуют sudo

**Решение:**
```bash
# Под root
usermod -aG docker deploy
# Затем перелогиньтесь под deploy
```

### Как посмотреть логи GitHub Actions

1. Откройте ваш репозиторий на GitHub
2. Перейдите на вкладку **Actions**
3. Кликните на нужный workflow run
4. Кликните на job "Deploy Bot"
5. Разверните шаги, чтобы увидеть логи

## Полезные команды

### На сервере (под пользователем deploy)

```bash
# Статус контейнеров
make status

# Логи бота (Ctrl+C для выхода)
make logs-bot

# Ручной перезапуск
make prod-stop
make prod

# Проверка здоровья
make ci-health
```

### Обновление вручную (без GitHub Actions)

```bash
cd ~/ВАШ_РЕПОЗИТОРИЙ
git pull origin main
make prod
```
